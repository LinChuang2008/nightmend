## 二、视觉一致性审计

### 2.1 色彩系统

**当前体系** (`src/styles/tokens.css`):

```css
--color-danger:  #ff4d4f   /* 严重告警 / 离线 */
--color-warning: #faad14   /* 警告 / 降级 */
--color-success: #52c41a   /* 健康 / 在线 */
--color-info:    #1677ff   /* 主色 / 信息 */
--color-ai:      #36cfc9   /* AI 特性专属 */
```

**优点**:
- 语义色彩与监控场景完美匹配（红=危险、橙=警告、绿=健康）
- AI 专属色 `#36cfc9` (Cyan) 明确区分 AI 功能层，差异化定位好
- 与 Ant Design `colorPrimary: #1677ff` 统一

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| C-1 | `tokens.css` 仅定义暗色主题变量，亮色模式无对应 CSS 变量 | P1 | 增加 `[data-theme='light']` 变量集，或统一使用 Ant Design token |
| C-2 | `statusColors.ts` 使用通用色名 (`'red'`, `'orange'`) 而非设计 token | P2 | 改为使用 CSS 变量或 Ant Design token 引用 |
| C-3 | `AIInsightBanner.tsx` 大量硬编码颜色 (`#0d1b2a`, `#1a2940`, `#8ab4cc` 等) | P2 | 提取 AI 主题色为 token 变量系列 (`--color-ai-bg-start`, `--color-ai-bg-end`) |
| C-4 | 图表配色 (CPU=#1677ff, Memory=#52c41a, Disk=#faad14) 与告警语义色重叠 | P2 | 资源指标应使用独立调色板，避免与告警色混淆 |
| C-5 | `serviceStatusText()` 硬编码中文 "健康"/"降级"/"异常" | P1 | 应使用 `t()` i18n 函数 |

### 2.2 字体规范

**当前设定**: 系统字体栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`

**评价**: 合理选择，跨平台渲染一致。

**建议**:
| # | 建议 | 优先级 |
|---|------|--------|
| F-1 | 仪表盘数字统计 (如 "2", "27", "69") 建议使用等宽数字特性 (`font-variant-numeric: tabular-nums`) 防止数字跳动时布局抖动 | P2 |
| F-2 | 侧边栏分组标签（"监控"、"日志与告警"等）字号 12px，与菜单项 14px 对比层级清晰，保持当前设定 | -- |

### 2.3 间距与布局

**当前体系**:
- 内容区 margin: 24px (桌面) / 12px (移动)
- 内容区 padding: 24px (桌面) / 16px (移动)
- 卡片间距: 12-16px (responsive)
- 仪表盘网格: 12 列, rowHeight 60px

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| S-1 | 服务器列表 (hosts.jpg) 只有 2 行数据，页面下半部全为空白，内容区高度未约束 | P2 | 空状态时收缩表格高度，或显示"无更多数据"提示 |
| S-2 | 数据库监控 (databases.jpg) 同上，3 行数据后大量空白 | P2 | 同上 |
| S-3 | 列表页表格间距宽松但列对齐良好，行高适中，保持当前 | -- | -- |

## 三、仪表盘设计审计

### 3.1 信息架构

**当前结构** (自上而下):
1. **Zone A**: AI 洞察横幅 + 健康评分仪表盘
2. **Zone B**: 5 个核心指标卡 (服务器/服务/数据库/活跃告警/异常日志)
3. **Zone C**: 服务器总览 + 趋势图 + 资源对比 + 日志统计 + 最新告警

**优点**:
- AI 洞察放在最顶部是正确的信息层级——AI 是产品差异化核心
- `react-grid-layout` 可拖拽调整+显示/隐藏，用户可定制程度高
- 健康评分仪表盘用 gauge 直觉表达整体状态
- 两阶段加载策略 (先核心指标后辅助数据) 优化了感知速度

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| D-1 | AI 洞察横幅在"暂无数据"时仍占据大片空间 (dashboard.jpg 所示) | P1 | 无数据时压缩为单行提示，有数据时再展开为完整横幅 |
| D-2 | 5 个指标卡缺少与前一时段的变化趋势（升/降/持平） | P1 | 每个卡片增加 sparkline 迷你趋势图或 delta 箭头 (如 "+3 ↑") |
| D-3 | 从截图看，AI 洞察横幅右侧有一个灰色装饰圆环，视觉上像未加载的图表 | P2 | 圆环脉冲动画需更明显或替换为更有意义的数据化装饰 |
| D-4 | 仪表盘页面标题"系统概览"后的 "●实时" 标记不够醒目 | P3 | 可加呼吸动画或放大为 badge |
| D-5 | 导出数据按钮和仪表盘设置位于右上角，但截图中不易辨识功能 | P3 | 考虑增加文字标签或 tooltip |

### 3.2 数据可视化

**图表选型评价**:
- 折线图用于趋势 (CPU/Memory/Alerts/Errors) — 正确
- 柱状图用于资源对比 — 正确
- 饼图用于日志级别分布 — 正确
- 仪表盘 Gauge 用于健康评分 — 正确
- 力导向图用于拓扑 — 正确

**ECharts 集成质量**: `ThemedECharts` 包装器自动适配暗色主题，全局一致性好。

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| V-1 | 主机详情页 (host-detail.jpg) 6 个图表布局紧凑合理，是全平台最佳数据展示页 | -- | 作为其他页面图表设计的标杆参考 |
| V-2 | SLA 页面 (sla.jpg) 3 个 100% 圆环图表现力不足，均为满值时视觉无差异 | P2 | 增加"当前月累计停机时间"数值和趋势比较 |
| V-3 | SLA 30 天趋势图 Y 轴范围 99%-100%，数据点聚集在顶部几乎看不出变化 | P2 | Y 轴起始点动态调整，或使用 "偏差视图" 突出与目标值的差距 |

**问题与建议**:

| A-1 | 告警标题被截断 ("服务不可用 - xiaoqia..."、"主机离线 - ecs-shan...") | P1 | 增加 tooltip 显示完整标题，或使用单元格内换行 |

**优点**:
- 顶部 4 个统计卡 (主机/服务/告警/错误日志) 提供快速上下文

- AI 对话区域采用问答列表式，可读性好

**问题与建议**:

| # | 问题 | 严重度 | 建议 |

## 五、服务拓扑图审计

### 5.1 当前实现

- **技术**: ECharts 力导向图 (`graph`)
- **节点分类**: 8 种类型 (Web/API/App/Registry/MQ/OLAP/Database/Cache)，每种独立颜色
- **边类型**: `calls` (实线蓝) / `depends_on` (虚线黄)
- **功能**: 节点拖拽、位置保存、编辑模式、AI 依赖建议、Pipeline 阶段视图

### 5.2 评价

**优点**:
- 颜色分类直觉且美观（8 色系来自暖/冷色调搭配）
- 编辑模式允许手动添加/删除依赖关系
- AI 依赖建议是差异化功能亮点
- Pipeline 分层视图 (Access → App → Middleware → Data) 提供了另一种理解架构的维度

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| T-1 | 力导向图在节点数 >20 时会混乱重叠，缺少自动布局优化 | P1 | 增加分层布局 (dagre) 选项，或按 Pipeline 阶段自动排列 |
| T-2 | 无缩放导航器 (minimap)，大型拓扑中容易迷失位置 | P2 | 增加右下角 minimap 组件 |
| T-3 | 节点状态（健康/异常）在拓扑图中不可见 | P1 | 用节点边框颜色或脉冲动画表示异常状态 |
| T-4 | 边标签（调用关系类型/延迟）不可见 | P3 | 鼠标悬停时显示调用详情 tooltip |

## 六、响应式设计审计

### 6.1 断点体系

```
xs: 0-480px     → 超小屏手机
sm: 480-768px   → 大屏手机
md: 768-1024px  → 平板
lg: 1024-1200px → 小桌面
xl: 1200px+     → 标准桌面
```

### 6.2 评价

**优点**:
- 移动端使用 `Drawer` 侧边栏，不占用内容空间
- 触摸目标最小 44px，符合 iOS HIG 和 Material Design 规范
- 表格自动水平滚动 (`overflow-x: auto`)
- 模态框自适应 (`max-width: calc(100vw - 32px)`)
- `useResponsive` 自定义 hook 统一管理断点

**问题与建议**:

| # | 问题 | 严重度 | 建议 |
|---|------|--------|------|
| M-1 | 移动端表格 `min-width: 600px` 强制最小宽度，小屏需频繁滚动 | P2 | 关键列表页提供卡片视图切换 (列表/卡片) |
| M-2 | Dashboard 网格在移动端是否自动调整为单列未明确 | P2 | 确保 `react-grid-layout` 在 xs/sm 断点下 cols=1 |
| M-3 | 暗色主题下 Drawer 使用 `theme: 'light'`，与主界面暗色风格不一致 | P1 | 移动端抽屉应跟随全局主题 |
| M-4 | 服务器列表/告警列表的操作按钮在窄屏可能被挤压 | P2 | 添加响应式操作列（窄屏 → 更多菜单） |
